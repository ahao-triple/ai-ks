# AI-KS 项目状态与功能清单

更新日期：2026-05-11

## 0. ECPM 看板重构 · Plan 1（基础 + 普通用户端）已落地

设计与计划文档：

- spec：`docs/superpowers/specs/2026-05-11-ecpm-dashboard-redesign-design.md`
- plan：`docs/superpowers/plans/2026-05-11-ecpm-dashboard-foundation-and-user.md`

本轮（Plan 1）实现：

- 后端 `RateLimitGuard` + `CachedResponseInterceptor`：针对 ECPM 看板这种纯查询接口的"静默节流"——超限请求不抛 429，而是标记 `rateLimitHit` 并由拦截器返回上次缓存（带 `X-Cache-Hit: true` 头）
- 后端 `UserDashboardModule`：提供 `/users/me/dashboard/{overview,groups,records}` 三个查询接口，覆盖 KPI 聚合 / 游戏与账号分组 / ECPM 单条记录（含"今日序号"语义）
- 前端 `UserDashboardPage`：普通用户登录后看到的新看板，包含 4 KPI、按"游戏 → 账号"分组的表格、ECPM 单条记录列表（自动刷新 + 限流静默 + 列设置）、移动端响应式
- 前端共享基础：`useThrottledRefresh` hook、`EcpmRecordTable` 通用组件、`AppShell` 左侧导航、`menusForScope` 按身份返回菜单项、`formatUserId/formatAccountId` ID 格式化
- 接入：账号视图（`activeView === 'account'`）现在先渲染 `UserDashboardPage`，旧 `AccountWorkspace` 继续保留在下方提供提现 / 资料 / open_id 绑定等功能；不破坏现有可用性

下一步（Plan 2）覆盖代理 / 公司管理员 / 超级管理员看板，以及 IA 全面切换到 `AppShell` 的左侧 10/4/3 项导航结构。

本文档是当前项目进度的主入口，用于替代早期 `docs/superpowers/specs` 和 `docs/superpowers/plans` 下的阶段性设计文档。旧文档已归档，只作为历史设计记录，不再代表当前实现状态。

## 1. 本轮对照旧文档发现的遗漏

这些遗漏来自旧规格、计划和当前代码实现的交叉核对。

### 1.1 已在本轮补齐

- 超级管理员端缺少公司管理员账号管理页面：已新增“权限”页，可创建公司管理员、启停账号、重置密码、分配公司和游戏可见范围。
- 超级管理员端所有操作堆在一个页面：已拆成“总览、联调、预算、授权、同步、结算、提现、审计、权限、维护”多个页面，并将高影响操作移入弹窗确认。
- 业务数据难以快速清理：已新增超级管理员“清空业务数据”能力，保留公司、游戏和管理员信息，清理联调产生的业务数据。
- 业务配置没有统一入口：已新增超级管理员“配置”页，可维护展示金额比例、用户/代理/默认代理/手续费分账比例、最低提现金额。
- 结算仍是展示金额全额入用户余额：已改为按平台配置生成分账快照，用户余额只入账用户比例，明细保留直属代理、上级代理、默认代理和手续费金额。
- ECPM 展示金额比例仍写死：已改为读取平台配置，新增同步按当前展示金额比例计算并写入快照。
- 结算批次缺少分账明细查看：已在超级管理员“结算”页新增“查看明细”，展示用户、代理、默认代理、手续费拆分。
- 默认代理无法在后台落地：已新增超级管理员“代理”页，可创建代理、查看代理余额，并把代理 ID 填入配置中心作为默认代理。
- 代理分账只停留在快照：已新增代理可用/冻结余额字段，结算确认时会把直属代理、上级代理、默认代理金额入账到代理可用余额。
- 代理余额无法进入提现流程：已支持超级管理员维护代理支付宝资料、发起代理提现，提现批次复用审核、模拟打款、失败关闭和余额退回流程。
- 代理端没有独立入口：已新增代理登录、JWT 鉴权、代理工作台，可查看代理资料、可用/冻结余额、分账收益明细、维护支付宝资料、自助提交提现、查看本人提现批次和名下用户。
- 用户归属代理没有闭环：已支持用户注册时填写代理邀请码、登录后换绑代理邀请码，并记录代理归属变更历史；后续结算按用户当前代理链路分账。
- 提现最低金额配置没有生效点：已接入用户提现服务，低于平台最低提现金额时拒绝申请。
- 后台按钮语义不够清楚：已补充操作说明、风险说明、字段辅助说明和确认弹窗，尤其覆盖预算、快手授权、结算、提现、权限、维护。
- 快手授权回调依赖手工复制：已支持从前端回调 URL 自动读取 `auth_code` / `code`，授权页展示回调地址格式。
- 快手 ECPM 失败任务无法复跑：已新增失败任务重试接口和前端按钮，按原任务数据小时范围重新拉取。
- 快手平台 secret 存储缺少保护：已支持 `KUAISHOU_TOKEN_ENCRYPTION_KEY` 加密保存数据库内的快手 secret，并兼容旧明文数据读取。
- 真实数据测试前缺少自动核对入口：已新增超级管理员“真实数据闭环核对”，可检查公司/游戏/预算、代理、用户代理归属、open_id、ECPM、结算和提现证据。
- 真实流程缺少操作手册：已新增 `docs/runbook/real-flow-debugging.zh-CN.md`。

### 1.2 仍待开发

- 代理体系已具备真实数据测试所需闭环：当前已完成超级管理员创建代理、默认代理配置、用户邀请码注册/换绑、代理归属历史、结算时代理余额入账、代理支付宝维护、代理提现申请、代理端登录、收益/提现视图和名下用户列表；待增强项主要是独立代理邀请落地页、二维码和推广素材。
- 配置中心仍需增强：已完成基础比例配置，仍缺少概率策略、封顶/最低/中包/高包模型、单笔提现上限、手续费承担方、平台收款支付宝、预警线等高级配置。
- 公司管理员目前是只读 MVP：还没有细粒度权限勾选、结算确认权、提现审核权、代理可见性权限、菜单能力元数据接口。
- 支付宝真实打款还没有接入：提现打款仍是模拟结果流程，缺少真实转账、失败重试、逐明细状态、人工处理标记和手续费明细。
- 快手授权仍需生产运维增强：已有回调读取、token 保存、授权码换 token、刷新、状态查询和 secret 加密存储，但缺少官方授权入口跳转、密钥轮换托管、授权异常恢复工作流。
- 快手 ECPM 同步任务还缺少运维增强：已有任务历史、失败任务重试和自动同步策略，但缺少队列化执行、分布式锁、任意时间范围补拉。
- 结算链路仍缺少高级场景：已有预览、确认入账、代理余额入账、代理/手续费快照和代理提现入口，但缺少预算暂停后的补结算工作台、公司管理员可审批的受控流程。
- 用户侧暂不扩展：当前只保留登录、收益查询、支付宝维护、提现申请等已有功能；密码找回、用户自助改密、更多账户安全能力未做。
- 报表和导出未做：没有财务报表、对账导出、数据仓库或 BI 图表。
- 生产发布治理未完善：需要补充正式迁移策略、密钥轮换、审计留存策略、备份恢复、真实外部接口压测。

## 2. 已落实且已有充分测试的功能

以下功能已有后端单元测试、前端单元测试或构建验证覆盖，可以作为后续真实流程调试的基础。

### 2.1 账号和权限

- 超级管理员登录、JWT 鉴权、超级管理员守卫。
- 公司管理员账号模型、登录身份识别、只读范围权限。
- 公司管理员 scope 管理：按公司和游戏授权可见范围。
- 公司管理员越权访问写接口返回 `403`。
- 前端超级管理员和公司管理员入口区分。

### 2.2 公司、游戏和预算

- 公司创建、列表、余额充值。
- 游戏创建、列表、配置保存。
- 游戏预算分配和预算余额展示。
- 游戏结算暂停开关。
- 公司管理员只看到授权范围内公司和游戏。

### 2.3 快手和游戏联调基础

- 游戏侧登录会话接口和 open_id 绑定基础流程。
- 快手 ECPM 手动刷新。
- 快手 ECPM 同步任务记录。
- 游戏级自动同步开关、同步间隔、最近执行和下次执行时间。
- 快手 token 状态查询、授权码换 token、刷新 token。
- 前端提供“联调、授权、同步”三个独立页面，便于真实游戏和快手流程排查。

### 2.4 结算和提现

- 平台配置中心：展示金额比例、用户结算比例、直属代理比例、上级代理比例、默认代理比例、手续费比例、最低提现金额。
- 超级管理员代理管理：创建代理、查看代理 ID、邀请码、上级代理、可用余额和冻结余额，维护代理支付宝资料并发起代理提现。
- 代理端工作台：代理账号登录、查看邀请码、可提现余额、冻结余额、代理收益明细、维护本人支付宝资料、提交本人提现申请、查看本人提现批次。
- ECPM 写入按平台展示金额比例计算展示金额，并把比例写入 `raw_ecpms.config_snapshot`。
- 管理员结算预览。
- 管理员确认结算并生成结算批次。
- 管理员确认结算时生成用户、直属代理、上级代理、默认代理、手续费拆分快照。
- 管理员确认结算时把直属代理、上级代理、默认代理金额入账到代理可用余额。
- 用户余额按“用户结算比例”入账，游戏预算按展示金额总额扣减。
- 结算批次列表、详情和前端分账明细查看。
- 用户支付宝收款信息维护。
- 用户提现申请，受平台最低提现金额限制。
- 代理提现申请，受平台最低提现金额限制，并冻结代理可用余额。
- 用户注册时可填写代理邀请码，登录后可换绑代理邀请码，代理归属历史会记录来源代理和目标代理。
- 代理工作台可查看直属用户和下级代理名下用户，用于核对真实数据结算前的归属关系。
- 管理员提现批次列表、详情、审核、模拟打款成功/失败、关闭失败批次。
- 失败代理提现关闭后会把冻结余额退回代理可用余额。

### 2.5 审计和真实数据核对

- 审计日志记录和后台查看。
- 启动时 Prisma schema 检查。
- 超级管理员真实数据闭环核对接口和总览页入口，用于真实联调前判断当前数据是否还有阻塞项。

### 2.6 前端体验

- 登录页、访客查询页、用户工作台、运营工作台。
- 运营工作台已拆分成页面式 tab，避免全部内容堆叠。
- 高影响操作使用弹窗确认，包括创建公司、充值、创建游戏、预算分配、快手授权、结算确认、提现操作、清空业务数据。
- 总览页聚合预算、同步异常、结算和近期数据。
- 权限页支持公司管理员创建、编辑和范围分配。
- UI 基础组件包含 Dialog、Panel、DataTable、MetricCard、Alert、StatusBadge。
