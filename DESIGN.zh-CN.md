# 前端视觉设计规范

## 目标

这份文档只定义前端的视觉方向和设计规则，不讨论具体重构步骤。

项目界面本质上是一个业务后台：用户会查看数据、管理账号、处理记录、审核提现。设计应该温暖、克制、清晰、可靠。我们保留原设计文档里“暖米色 + 珊瑚色 + 深墨色”的气质，但把它从营销官网风格简化成适合长期操作的后台系统。

## 组件与样式选择

新前端不再使用 HeroUI。

推荐组合：

- **shadcn/ui**：作为组件基础。
- **Tailwind CSS**：负责样式和设计 token。
- **Radix UI**：通过 shadcn/ui 提供可访问的底层交互。
- **lucide-react**：统一图标。
- **TanStack Table**：复杂表格场景使用。
- **react-hook-form + zod**：表单和校验场景使用。

选择 shadcn/ui 的原因是它不强迫项目接受固定视觉风格，适合做一套自己的暖色业务后台。

## 设计方向

整体方向是：**温暖的业务后台极简风格**。

关键感受：

- 背景偏暖，不使用纯白或冷灰作为页面主底色。
- 珊瑚色只用于主操作、当前选中、关键确认，不铺满页面。
- 文字使用接近黑色的暖墨色，避免刺眼。
- 页面以表格、筛选、状态、操作为中心。
- 用边框和浅底色分层，少用阴影。
- 间距紧凑但不拥挤，适合反复操作。
- 控件圆角控制在 6-10px，避免过度圆润。
- 动效只服务于反馈，不做装饰。

不要做营销页式的大 hero、大留白、渐变背景、装饰性卡片堆叠。

## 色彩

推荐核心色板：

```css
:root {
  --background: #faf9f5;
  --foreground: #141413;

  --surface: #ffffff;
  --surface-soft: #f5f0e8;
  --surface-muted: #efe9de;

  --border: #e6dfd8;
  --border-strong: #d8cec1;

  --muted: #6c6a64;
  --muted-soft: #8e8b82;

  --primary: #cc785c;
  --primary-hover: #a9583e;
  --primary-soft: #f4ded5;

  --success: #4f9f65;
  --success-soft: #e4f3e8;
  --warning: #d4a017;
  --warning-soft: #fbf0cc;
  --danger: #c64545;
  --danger-soft: #f7dddd;

  --info: #4f83a8;
  --info-soft: #e3eef5;

  --dark-surface: #181715;
  --dark-surface-elevated: #252320;
  --on-dark: #faf9f5;
}
```

### 使用规则

- `background`：应用外壳和页面背景。
- `surface`：表格、面板、弹窗、表单、下拉菜单。
- `surface-soft`：筛选区、浅色分组区域、页面局部分隔。
- `surface-muted`：选中态、弱提示、空状态背景。
- `primary`：主按钮、当前导航、选中标签、关键确认。
- `primary-soft`：选中行、轻提示、非危险的重点区域。
- `danger`：删除、拒绝、失败、校验错误。
- `success`：通过、完成、启用、已支付。
- `warning`：待审核、警告、异常但未失败。
- `info`：处理中、普通系统提示。
- `dark-surface`：少量重点摘要区域，不作为常规背景。

一个页面大部分时候应该是暖米色背景、白色面板、深色文字和浅边框。珊瑚色只在需要用户注意的位置出现。

## 字体

后台界面不强制品牌字体，不使用复杂的衬线标题。

推荐字体：

```css
font-family:
  Inter, ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont,
  "Segoe UI", sans-serif;
```

推荐字号：

| 名称 | 大小 | 字重 | 用途 |
| --- | ---: | ---: | --- |
| 页面标题 | 24px | 650 | 页面主标题 |
| 区块标题 | 18px | 600 | 面板、表格标题 |
| 正文 | 14px | 400 | 默认界面文字 |
| 强调正文 | 14px | 600 | 表格重点字段 |
| 辅助文字 | 12px | 500 | 说明、元信息、标签 |
| 按钮文字 | 14px | 600 | 按钮 |

除小号大写标签外，字间距保持 `0`。

## 布局

### 应用框架

后台使用稳定的管理台布局：

- 桌面端左侧边栏作为主导航。
- 顶部栏显示页面上下文、账号操作和全局控制。
- 主内容区使用暖米色背景。
- 内容面板使用白色背景和浅边框。
- 移动端侧边栏收起为抽屉。

### 页面结构

常规页面顺序：

1. 页面标题区：标题、简短说明、主操作。
2. 可选的数据摘要。
3. 筛选和搜索区域。
4. 主表格或列表。
5. 详情抽屉或编辑弹窗。

第一屏应该直接可用，不做营销页式介绍。

### 间距

使用 4px 作为基础单位。

- 页面边距：桌面 24px，平板 16px，移动端 12-16px。
- 面板内边距：16-20px。
- 表单行间距：12-16px。
- 表格单元格：上下 10-12px，左右 12-16px。
- 工具栏控件间距：8-12px。

后台页面不使用 64-96px 的大段营销留白。

## 形状、边框和层级

- 默认圆角：8px。
- 小控件：6px。
- 弹窗和主要面板：10px。
- 标签和胶囊元素：999px。
- 常规分隔使用 `1px solid var(--border)`。
- 少用阴影，优先用边框、底色和间距建立层级。
- 不要把卡片套在卡片里，除非是弹窗里的列表项这类明确结构。

## 组件视觉规则

### 按钮

按钮按意图区分：

- 主按钮：珊瑚色填充，白色文字。只用于当前页面最重要的动作。
- 次按钮：白色或透明背景，带边框。
- 幽灵按钮：透明背景，用于低强调工具操作。
- 危险按钮：红色填充或红色文字，根据危险程度决定。
- 图标按钮：使用 lucide 图标，并提供可访问标签。

高度建议：

- 默认按钮：36-40px。
- 表格紧凑操作：32px。
- 移动端主要操作：至少 40px。

### 导航

侧边栏视觉：

- 当前项使用 `primary-soft` 背景和珊瑚色文字。
- 不要每个选中态都使用大面积纯珊瑚色。
- 图标统一使用 lucide，尺寸 18-20px。

导航应该安静，不要抢表格和内容的注意力。

### 表格

表格是这个系统的核心组件。

- 表头使用 `surface-soft` 或白底加底边框。
- 长表格优先使用 sticky header。
- 行 hover 使用 `surface-soft`。
- 选中行使用 `primary-soft`。
- 金额、数量、比例、日期使用更稳定的数字显示。
- 需要比较的数字列可以右对齐。
- 操作列保持紧凑和固定位置。
- 状态使用标签，不直接显示数字状态。

移动端密集表格可以横向滚动。提现审核这类流程，如果卡片列表更清晰，也可以使用卡片列表。

### 筛选和搜索

筛选区要实用、紧凑：

- 使用白底或 `surface-soft`。
- 圆角 8px。
- 桌面端尽量单行。
- 平板自然换行。
- 移动端可堆叠或进入筛选抽屉。

重要筛选项必须有明确标签，不只依赖 placeholder。

### 表单

表单应该清楚、安静：

- 标签在输入框上方。
- 帮助文本或错误文本在输入框下方。
- 聚焦状态使用低透明度珊瑚色描边或光环。
- 必填状态通过文字和校验体现，不用强烈红色装饰。
- 相关字段用小标题和边框分组，不使用大装饰卡片。

### 弹窗和抽屉

短确认用弹窗，信息较多的详情审核用抽屉。

- 弹窗背景使用白色。
- 遮罩使用 35-45% 黑色透明度。
- 头部、内容、底部操作区要分清楚。
- 危险确认要有明确红色样式。
- 提现通过或拒绝前，必须清楚展示用户、金额、收款身份。

### 状态标签

状态颜色保持语义一致：

- 待审核：浅黄色背景，黄色文字。
- 已通过/已完成：浅绿色背景，绿色文字。
- 已拒绝/失败：浅红色背景，红色文字。
- 停用/无效：浅米色背景，灰色文字。
- 处理中/信息：浅蓝色背景，蓝色文字。

标签要小而清晰，不使用大面积高饱和填充。

### Toast 提示

Toast 内容要短，直接说明结果：

- 成功：动作已完成。
- 失败：动作失败和可读原因。
- 警告：已完成但有注意事项。
- 信息：后台任务已开始。

重要表单错误不能只放在 Toast 中，也要在字段附近显示。

### 空状态和加载状态

- 空状态说明当前缺什么，并给出下一步动作。
- 表格和面板加载优先用 skeleton。
- 除非整个应用不可用，否则不要使用全屏 loading。

## 数据和金额展示

- 金额小数位保持一致。
- 日期格式保持统一。
- 长 ID 可截断，并在需要时提供复制。
- 敏感信息默认遮罩，除非当前任务必须展示。
- 颜色用于表达状态，不随意装饰数字。

## 响应式

断点：

- 移动端：小于 768px。
- 平板：768-1024px。
- 桌面：1024px 以上。

规则：

- 移动端侧边栏变成抽屉。
- 表格可以横向滚动。
- 关键操作按钮必须容易触达。
- 弹窗在移动端接近全宽。
- 文字不能溢出按钮、标签、表格单元格。
- 不要用视口宽度动态缩放字体。

## 可访问性

- 保留 Radix/shadcn 的键盘交互能力。
- 只有图标的按钮必须有可访问标签。
- 聚焦状态必须可见。
- 暖色背景上的文字对比度必须足够。
- 不只用颜色表达状态，要配合文字或图标。
- 危险操作需要确认，能撤销的尽量提供撤销。

## 应该做

- 统一使用暖色调。
- 页面以任务和表格为中心。
- 用边框、间距和字体层级建立结构。
- 操作按钮靠近对应数据。
- 适度使用 lucide 图标提升扫描效率。
- 复杂记录详情优先用抽屉。
- 保持界面安静，适合长时间操作。

## 不应该做

- 不复刻原来的 Claude 营销页风格。
- 不在后台里做大 hero。
- 不过度使用珊瑚色。
- 不使用紫蓝渐变、装饰光斑等背景。
- 不使用重阴影和嵌套装饰卡片。
- 不让每个页面都有完全不同的视觉。
- 不把关键操作藏在 hover 里。
- shadcn/ui 已经能满足时，不优先自造复杂组件。

## 给开发时的落地建议

做界面时按这个顺序：

1. 先选 shadcn/ui 组件。
2. 用 Tailwind theme 或 CSS variables 接入上面的色彩 token。
3. 至少两个页面重复出现同一模式后，再抽公共组件。
4. 主要组件语言是表格、筛选条、弹窗、抽屉、状态标签、紧凑表单。
5. 每次前端修改后，检查桌面和移动端是否有文字溢出、控件重叠、操作不可点的问题。

如果具体需求和这份规范冲突，优先保证任务清晰、操作安全、信息可读。
