# 真实流程联调 Runbook

更新日期：2026-05-10

本文档用于游戏接入和快手 ECPM 联调。结算和真实打款不在本轮联调范围内，最多只做到结算预览前后的数据核对。

## 1. 启动和账号

本地启动：

```bash
pnpm dev
```

当前常用地址：

- 前端：`http://127.0.0.1:8012/`
- API：`http://localhost:8007`

默认超级管理员来自 `.env`：

- `ADMIN_USERNAME=admin`
- `ADMIN_PASSWORD=admin123456`

如果改过 `.env`，以本地 `.env` 为准。

## 2. 联调前清理

进入超级管理员端后打开“维护”页，执行“清空测试数据”。

清空会删除：

- 公司、游戏、用户和 open_id 绑定。
- 快手平台 token、ECPM 同步任务。
- 平台业务配置、预算、结算、提现和审计日志。

真实联调环境不再提供页面上的测试数据清空入口。需要重置数据时，应直接对测试数据库做备份后清理或重建。

## 3. 快手真实模式配置

真实快手接口需要设置：

```bash
KUAISHOU_API_MODE=real
KUAISHOU_TOKEN_ENCRYPTION_KEY=
```

`KUAISHOU_TOKEN_ENCRYPTION_KEY` 建议使用稳定的 32 字节 base64 值。变更该值后，数据库里已经加密保存的快手 secret 将无法解密。

如果暂时不走后台授权，也可以用环境变量兜底：

```bash
KUAISHOU_ACCESS_TOKEN=
KUAISHOU_ADVERTISER_ID=
```

优先级：数据库授权 token 高于环境变量兜底。

## 4. 创建联调资源

先打开“配置”页，确认基础业务口径：

1. 展示金额比例：影响后续 ECPM 写入的展示金额。
2. 用户、直属代理、上级代理、默认代理、手续费五项分账比例：合计必须为 100%。
3. 最低提现金额：低于该金额的提现申请会被拒绝。

如果默认代理比例大于 0，或希望缺失代理的份额有明确归属，先打开“代理”页创建代理，再把代理 ID 填入“配置”页的默认代理 ID。代理页也可以维护代理支付宝资料并发起代理提现，提现后到“提现”页继续审核和模拟打款。

代理账号创建后，可退出超级管理员端，在登录页切换到“代理”入口，用代理用户名和密码进入代理工作台。代理工作台可查看本人收益、余额、支付宝资料和提现批次，适合核对结算后代理侧是否同步入账。

用户侧归属代理有两种方式：

1. 注册用户时填写代理邀请码。
2. 用户登录后进入“账号工作台 -> 代理归属”，填写新的代理邀请码完成换绑。

后续新结算按用户当前代理链路分账；已经生成的历史结算和提现批次不回改。

如果只是验证游戏和快手联调，可以先使用默认配置。

在“预算”页按顺序执行：

1. 创建公司。
2. 给公司余额充值。
3. 创建游戏，填写 `game_app_id` 和 `game_secret`。
4. 给游戏分配预算。

游戏创建后会出现在“联调”页和“预算”页。

## 5. 总览闭环核对

打开“总览”页，找到“真实数据闭环核对”，点击“刷新核对”。

开始真实数据测试前至少要处理所有 `阻塞` 项：

- 公司 / 游戏 / 预算：必须有公司、游戏和可用游戏预算。
- 代理账号：至少创建一个启用代理；如果要验证默认代理分账，把代理 ID 写入“配置”页。
- 用户代理归属：真实测试用户需要通过邀请码注册，或登录后在“代理归属”里换绑。
- open_id 绑定：真实游戏登录后产生的 open_id 需要绑定到用户。
- ECPM 收益明细：至少要有待结算 ECPM，或已经生成过结算批次。

`需关注` 不一定阻止游戏和快手联调，但用于提醒后续闭环还没完整跑完。若本次目标是业务闭环验收，应继续把结算和提现也跑到 `已就绪`。

## 6. 快手授权

打开“授权”页。

回调地址格式：

```text
http://127.0.0.1:8012/?auth_code=AUTH_CODE&app_id=APP_ID
```

也兼容：

```text
http://127.0.0.1:8012/?code=AUTH_CODE
```

快手授权平台回跳到前端后，页面会自动读取 `auth_code` 并填入授权表单。确认 `app_id` 和 `secret` 后，点击“提交授权”。

授权成功后检查：

- 状态为 `ACTIVE`。
- 显示广告主 ID。
- 显示 access token 和 refresh token 过期时间。
- 页面不展示 access token、refresh token 或 secret 明文。

## 7. 游戏登录联调

打开“联调”页。

步骤：

1. 选择游戏。
2. 输入真实游戏侧传来的 `js_code`。
3. 点击“换取 open_id”。
4. 确认返回可读 ID 和 open_id。

如果失败，先确认：

- 游戏 `game_app_id` 和 `game_secret` 与游戏侧一致。
- API 服务正在运行。
- 该游戏没有被清空测试数据操作删除。

## 8. ECPM 同步联调

打开“联调”页或“预算 -> 游戏配置 -> ECPM 同步”。

手动刷新：

1. 选择游戏。
2. 选择回看范围，支持 1/3/6/12/24 小时。
3. 点击“刷新游戏 ECPM”或“手动刷新 ECPM”。
4. 检查写入条数、来源和明细。

任务排查：

1. 打开“同步”页。
2. 点击“刷新任务”。
3. 优先查看 `FAILED` 任务的错误信息。
4. 授权过期时先回到“授权”页刷新 token。
5. 问题修复后点击失败任务行里的“重试失败任务”。

重试会按失败任务记录的开始/结束数据小时重新拉取同一范围，不会简单改成“最近 N 小时”。

## 9. 最小验收

一次游戏和快手联调至少要留下这些证据：

- 一个公司和一个游戏。
- 一个通过游戏登录换到的 open_id。
- 一个成功的快手授权状态。
- 一个 `SUCCEEDED` 的 ECPM 同步任务。
- 刷新明细中能看到该 open_id 的 ECPM 数据。
- “总览 -> 真实数据闭环核对”没有 `阻塞` 项。
- 审计日志能看到授权、刷新或失败重试相关记录。

如果同时验证业务闭环，还需要补充：

- “配置”页比例保存成功。
- “结算”页预览后确认入账，批次详情能看到用户、代理、默认代理和手续费拆分。
- 用户余额只增加用户结算比例对应金额。
- “代理”页能看到直属代理、上级代理或默认代理的可用余额增加。
- 代理端登录后能看到同一笔直属、上级或默认代理分账收益。
- 代理端“代理名下用户”能看到注册或换绑到该代理的用户；上级代理也能看到下级代理名下用户。
- 代理页发起代理提现后，代理可用余额减少、冻结余额增加，“提现”页出现 `AGENT` 类型批次。
- 低于最低提现金额的提现申请会被拒绝。

## 10. 常见失败处理

- API 连接失败：确认 API 端口和 `VITE_API_BASE_URL` 一致。
- 授权失败：确认 `app_id`、`secret`、`auth_code` 属于同一授权周期。
- token 解密失败：确认 `KUAISHOU_TOKEN_ENCRYPTION_KEY` 与写入 token 时一致。
- ECPM 返回为空：确认 open_id 已通过游戏登录写入，并且所选数据小时确实有快手收益。
- ECPM 失败任务反复失败：先刷新 token，再重试失败任务；仍失败时查看任务错误和审计日志。
